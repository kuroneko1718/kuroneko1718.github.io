Ubuntu

虚拟机问题

  • 虚拟机下载
    • www.virtualbox.org/wiki/Downloads
  • 系统镜像CD安装
    • 虚拟机屏幕显示问题。
      1. 因为虚拟机分辨率问题，安装界面出现显示不全问题，导致安装界面下方的按钮无法显示，可以使用Alt+F7拉动界面，并用鼠标右键固定
      2. 因为虚拟机的显示分辨率的问题，可以使用按键Ctrl+Alt+T进入终端，并输入xrandr -s 1280x800 修改屏幕分辨率。
    • 虚拟机硬盘分区问题
  • 虚拟机网络问题
    • 虚拟机安装成功之后网络连接失败
      1. vmware network adapter vmnet8 没有有效的IP配置，因为设置时为自动获取IP，也就是要使用DHCP服务。宿主机为win10，需要开启服务，VMware NAT service和VMware DHCP service服务 并使其自动运行。
  • 虚拟机镜像源问题
    • 安装成功之后无法使用sudo apt-get 安装应用。
      1. 可通过修改ubuntu镜像源解决。
  • 装机应用
    • vim, net-tools(ifconfig, etc…), iptables, git, python3, java, golang, node, php, ruby, rust, docker, nginx, mysql, tomcat, vsftpd
  • 虚拟机开启ssh22端口映射
    • 先安装net-tools，然后实用ifconfig命令(或者ip addr)查看虚拟机网卡IP地址配置(选择以太网网卡IP)。选择虚拟机的外网网卡ip地址之后，在虚拟机软件(VirtualBox)的网络设置中添加对应的端口配置信息，实用NAT端口转发(高级)填写规则。再重启虚拟机
      名称 协议 主机IP 主机端口 子系统IP 子系统端口
      ssh tcp 127.0.0.1 22022 10.0.2.15 22
  • 开启ssh登录(局域网)
    • 安装openssh-server sudo apt-get install -y openssh-server
    • 查看ssh状态 sudo systemctl status ssh
    • ufw(ubuntu防火墙)开放ssh端口sudo ufw allow ssh
  • 客户机ssh登录ubuntu
    • 客户机使用ssh登录
    • 免输入密码登录
      1. 先在服务机开启ssh服务，并打开对应端口号，并放行
      2. 在服务机当前用户目录创建.ssh 目录用来存放客户机的公钥文件id_rsa.pub的authorized_keys文件
      3. 在客户机当前用户目录创建.ssh 目录用来存放要生成ssh连接的公钥(id_rsa.pub)和私钥(id_rsa)文件
      4. 把客户机生成的id_rsa.pub公钥文件复制到服务机的authorized_keys中，一行一个
      5. 再登录时不会再提示要输入密码了
  • 端口操作
    • 查看端口开放
      1. sudo netstat -atnlp
      2. sudo lsof -i:22(对应端口号)
    • 开放端口（需安装应用，iptables, iptables-persistent）
      1. sudo iptables -I INPUT -p tcp –dport 80 -j ACCEPT
      2. sudo iptables-save # 保存
      3. sudo iptables-persistent save # 持久端口开放操作，设置为自启动(netfilter-persistent)
      4. sudo iptables-persistent reload # 重启
  • mysql相关
    • 安装mysql，虚拟机存储问题不能成功安装
      1. 给虚拟机虚拟磁盘扩容，打虚拟机软件（VirtualBox）全局设置-> 虚拟机介质管理，调整对应的虚拟机虚拟磁盘修改大小。
      2. 登录虚拟机，打开虚拟机存储管理器(disk)，先修改对应扩展分区大小在修改文件系统分区大小。
    • 宿主机连接虚拟机mysql
      1. root用户登录之后，创建一个新用户授予同root用户权限用在宿主机数据库客户端软件(dbeaver, navicat)，通过ssh方式登录。参考mysql.md

段落与换行

  • 段落由一行或多行文本组成，不同的段落之间使用空行来标记，同层级的段落或行需要加空行来声明, 或者在段落或行之后加三个空格，
    1. 如果行与行之间没有空行，则会被视为同一段落。
    2. 如果行与行之间有空行，则会被视为不同的段落。
    3. 空行是指行内什么都没有，或者只有空格和制表符。
    4. 如果想在段内换行，则需要在上一行的结尾插入两个以上的空格然后按回车键。
  • 使用规范：为了便于阅读，应该限制每行字符的数量，通常每行不超过80个字符，可以在编辑器中进行设置。
  • 关于换行
    1. 当超过80个字符后进行换行。
    2. 在一句话结束（。或!或?）之后换行。
    3. 当URL较长时换行。通常URL较长会导致行字符数量超过限制，为了提高可读性，可以在URL之前加一个换行符。

超链接

  • 下面是github的网址
    Github
  • 引用链接
    • 把链接地址在某个地方统一定义好，然后在正文中通过“变量 ”来引用，可读性一下子就变强了，这种方法叫作引用链接。
    • 引用链接是把链接地址作为“变量 ”先在Markdown文件的页尾定义好，然后在正文中进行引用。
    • 链接标记可以有字母、数字、空格和标点符号。
    • 链接标记不区分大小写。
    • 定义的链接内容可以放在当前文件的任意位置，建议放在页尾。
    • 当链接地址为网络地址时要以 http/https开头，否则会被识别为本地地址。
  • 网址链接
    • 在Markdown中，将网络地址或邮箱地址使用<>包裹起来会被自动转换为超链接。
  • 使用规范
    • 在Markdown中，链接标题的信息应该更丰富，从标题中应该可以知道链接的内容，要使用有意义的链接标题。
    • 建议使用<>包裹自动链接，这种方式更通用。
    • 自动链接要以http/https开头。

列表

  • 无序列表
    无序列表均可用字符 +-* 来创建（序号与内容之间添加三个空格~）
  • 有序列表
    1. 数字加小数点（序号与内容之间添加三个空格）
  • 列表嵌套
    • 二级列表 （+-*）
    1. 下级列表前面需添加三个空格
  • 使用规范
    • 建议使用 - 来标记无序列表，因为 * 容易跟粗体和斜体混淆，而 + 不流行。
    • 如果一个列表中所有的列表项都没有换行，建议使用1个空格。
    • 如果列表项有换行，则建议给无序列表使用3个空格，给有序列表使用2个空格。
    • 如果一个列表中的每个列表项都只有1行，建议列表项之间不要有空行。
    • 数字、字符、符号列表使用英文半角句号，句号后加空格。

表格

  • 设置表格对齐，在第二行的表格标识中设置
  • 块级元素（代码区块、引用区块）不能插入表格中。
  • 使用规范
    1. 在表格的前、后各空1行。
    2. 在每一行最前和最后都使用|，每一行中的|要尽量都对齐。
    3. 不要使用庞大复杂的表格，那样会难以维护和阅读。
      表头 表头 表头
      默认靠左居中 设置中间居中 设置靠右居中

代码

  • 单行代码 console.log('test'); //单行代码
  • 多行代码
  • 很多Shell命令都要粘贴到终端中去执行，因此最好避免在Shell命令中使用任何换行操作；可以在行尾使用一个\，这样既能避免命令换行，又能提高源码的可读性。
  • 建议不要在没有输出内容的Shell命令前加$。在命令没有输出内容的情况下，$是没有必要的，因为内容全是命令，我们不会把命令和输出的内容混淆。
  • 建议在有输出内容的Shell命令前加上$，这样会比较容易区分命令和输出的内容。
